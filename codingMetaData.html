Coding and Metadata: Tables and Forms

All of the labels on your HTML have meaning, and those meanings should be regarded first and foremost. Your h1, h2, h3 etc. should be heirarchical, and unordered lists vs. ordered lists should be used with intention. This applies for all of your structured HTML, and is essential for any user.

There's also a great guide on how to implement ARIA, which is a method of adding information to your HTML so that disabled users are able to navigate more complex user interactions from W3C, which you can check out here:

ARIA Guide!

Forms

For forms, one of the most important things is to code for user error. Make it impossible for users to enter invalid answers, and let them know what is invalid about their answers as well. Make sure that your forms are tabbable, and that they have clear color contrasts. Clear input types are essential!

Tables

Tables are all about semantics. Make sure you create a semantic table, where all portions of the table are clearly labled with semantic HTML. This is relatively simple; use labels such as thead, tbody, etc.
